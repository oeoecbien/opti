---
### 1. LA PROGRAMMATION LINÉAIRE (PL) ###
---

# CONCEPT :
# C'est une technique mathématique pour trouver le meilleur résultat (profit max, coût min...)
# dans un modèle dont toutes les relations sont linéaires.

# INGRÉDIENTS OBLIGATOIRES POUR MODÉLISER UN PROBLÈME :
# a) Variables de décision (ex: x1, x2): Les inconnues que l'on cherche, les leviers sur lesquels on peut agir.
#    - "Combien de produit A fabriquer ?" -> x_A
# b) Fonction objectif (ex: Z): L'équation mathématique qui représente notre but. Doit être maximisée ou minimisée.
#    - "Maximiser Z = 10*x_A + 15*x_B"
# c) Contraintes : Les limites, les règles du jeu. Ce sont des inéquations.
#    - "On a 120h de travail max" -> 2*x_A + 3*x_B <= 120
#    - "On ne peut pas produire une quantité négative" -> x_A >= 0, x_B >= 0

---
### 2. LA MÉTHODE DU SIMPLEXE ###
---

# CONCEPT :
# C'est l'algorithme (la "recette de calcul") qui résout un problème de programmation linéaire.
# La méthode graphique est limitée à 2 variables, le simplexe fonctionne pour des milliers.

# COMMENT ÇA MARCHE (VULGARISÉ) :
# 1. La zone des solutions réalisables (définie par les contraintes) est un polygone (ou polyèdre en N dimensions).
# 2. La théorie nous dit que la solution optimale se trouve FORCÉMENT sur un des sommets de ce polygone.
# 3. L'algorithme du simplexe est un robot intelligent :
#    - Il part d'un sommet.
#    - Il regarde les sommets voisins.
#    - Il se déplace vers le voisin qui améliore le plus la fonction objectif.
#    - Il répète l'opération jusqu'à ce qu'il n'y ait plus de voisin meilleur. Il a alors trouvé l'optimum.

# EN PRATIQUE :
# On n'applique jamais le simplexe à la main. On utilise des logiciels (solveurs) ou des bibliothèques
# comme PuLP en Python, qui l'implémentent pour nous.

---
### 3. LA DUALITÉ EN PROGRAMMATION LINÉAIRE ###
---

# CONCEPT :
# Chaque problème de PL (le "Primal") possède un problème "miroir" (le "Dual").
# Ils sont intimement liés. Si le Primal est une maximisation, le Dual est une minimisation, et vice-versa.

# THÉORÈME FONDAMENTAL :
# À l'optimum, la valeur de la fonction objectif du Primal est EXACTEMENT ÉGALE à celle du Dual.
# C'est un excellent moyen de vérifier un résultat.

# RECETTE POUR FORMER LE DUAL (à partir d'un Primal en minimisation) :
# 1. Créer une variable de dualité (y1, y2...) pour chaque contrainte du Primal.
# 2. La fonction objectif du Dual est une MAXIMISATION. Ses coefficients sont les seconds membres (la partie droite) des contraintes du Primal.
# 3. Les contraintes du Dual (une par variable du Primal) ont pour second membre les coefficients de l'objectif du Primal.
# 4. La matrice des coefficients des contraintes est "transposée".

---
### 4. L'OPTIMISATION MULTI-OBJECTIFS ###
---

# CONCEPT :
# C'est ce qu'on utilise quand on a plusieurs objectifs, souvent contradictoires (ex: maximiser la qualité ET minimiser le coût).
# Il n'y a plus UNE solution optimale, mais un ENSEMBLE de bons compromis.

# LE FRONT DE PARETO :
# C'est l'ensemble de toutes les solutions "non-dominées".
# Une solution A "domine" une solution B si A est meilleure ou égale à B sur tous les objectifs,
# et strictement meilleure sur au moins un.
# Le Front de Pareto contient donc tous les meilleurs compromis possibles.

# COMMENT CHOISIR UNE SOLUTION DANS LE FRONT DE PARETO ?
# On utilise une méthode d'aide à la décision pour classer les solutions du front.

# a) Méthode de la Somme Pondérée (ou Agrégée) :
#    - Principe : On fusionne tous les objectifs en un seul score en leur attribuant des poids (w1, w2...).
#    - Formule : Score = w1 * Obj1_norm + w2 * Obj2_norm + ...
#    - ÉTAPE CRUCIALE : Il faut d'abord NORMALISER les valeurs des objectifs (les ramener sur une échelle commune, souvent 0-1)
#      pour qu'ils soient comparables.

# b) Méthodes de Sur-classement (ex: ELECTRE, PROMETHEE) :
#    - Principe : Au lieu de calculer un score unique, on compare les solutions deux à deux.
#    - On construit une "relation de sur-classement" ("La solution A est-elle préférable à B ?") en se basant sur les
#      préférences et les poids des critères.
#    - Résultat : On obtient un classement (partiel ou total) des solutions, ce qui aide le décideur à choisir.
